Bonne gestion de droits sur un serveur de fichiers Microsoft:

 

L'objectif est de bien structurer les permissions (d’avoir des groupes avec des noms bien clairs) optimiser et renforcer la sécurité au sein du domaine

 

Les types de groupes :

Sécurité :

Ce type de groupe gère les droits pour donner les autorisations d’accès aux ressources (partages de dossiers, de fichiers, imprimantes bureau à distance etc.)

Distribution :

Principalement utilisés pour les applications de messagerie et n’a aucune notion de sécurité pour les envois multiples de mails par exemple, on pourra diffuser le mail à toutes les personnes présente dans le groupe de diffusion.

Cette gestion prend son sens si on a une infra très étendue. (Relation d’approbation, création d’un nouveau domaine…)

Un utilisateur sera membre d’un groupe global par exemple un groupe de comptables, et ce groupe sera lui-même membre d’un groupe local qui gèrera les accès et lecture, écriture sur les partages de dossiers cet utilisateur aura donc accès à certains types de fichiers

 

Principe d’imbrication de groupe

On va gérer les utilisateurs avec des groupes globaux (RH, compta, direction

Un groupe global peut être utilisé dans un autre domaine et permet plus de possibilité on part du principe qu’aucun user ne doit être membre global uniquement

 

On gère ensuite les accès sur les partages de dossiers et fichiers en lecture et écriture des groupes globaux avec des « sous » groupes de domaines locaux.

Un groupe domaine local pourra être utilisé uniquement dans le domaine active directory dans lequel il est créé

 

Le groupe universel ajoute une couche de sécurité en plus avant tout, il est accessible dans l’ensemble de la forêt très peu de personnes ont des accès uniquement universels, ils seront gérés de la même manière pour confiner l’accès à certaines zones du domaine.
