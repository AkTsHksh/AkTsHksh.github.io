DNS (Domain Name System)

Il permet d’associer un nom compréhensible a une adresse IP, il associe une adresse logique (le nom de domaine) à une adresse physique (IP) un peu à la manière d’un annuaire téléphonique (Et vice-versa).Il est utilisé pour localiser les serveurs de domaines Active Directory.

Il utilise les protocoles UDP & TCP et le port 53

 

COMMENT ?

  • Le LDNS (Local DNS) s’adresse à l’un des serveurs racine qui le dirige vers le serveur DNS.com
  • Le serveur DNS.com découvre ensuite le propriétaire du site recherché et notifies le LDNS avec un enregistrement de serveur de nom (NS) pour le site concerné
  • Le LDNS répond en demandant un enregistrement d’Adresse (enregistrement A) qui inclus l’adresse IP du site en question.
  • Après avoir reçu l’enregistrement A, le LDNS enverra l’adresse IP au navigateur et met en cache les informations d’adresses IP pour référence future. (L’enregistrement A et utilisé par les IPv4 l’enregistrement AAAA est pour les IPv6)

Il élimine la nécessité de mémoriser les adresses IP et utilise le nom de domaine pour l’adresse web.

Chaque appareil et service sue le World Wide Web possède une IP, n’étant pas facile à retenir, le DNS fournit une solution à ce problème en associant un nom de domaine à une adresse IP.
