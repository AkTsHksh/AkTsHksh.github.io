Création d'un sysprep

CREATION D’UNE IMAGE WINDOWS 10 PROFESSIONNEL PERSONALISEE VIA SYSPREP

 

 

Nous allons maintenant créer une nouvelle machine virtuelle sous Windows 10 professionnel pour créer notre système d’exploitation préconfiguré à capturer sur WDS. (BIEN FAIRE ATTENTION A SELECTIONNER WINDOWS 10 PROFESSIONNEL !!)

 

Une fois installée on y ajoute les programmes désirés sur l’image que l’on souhaitera déployer ici j’y ai ajouté les TOOLS comme j’utilise VMWare pour ne pas avoir à le refaire à chaque fois, PUTTY et google chrome.

On effectue le sysprep en OOBE, en cochant la case « Généraliser » et en Arrêtant le système.

 

Une fois installée on y ajoute les programmes désirés sur l’image que l’on souhaitera déployer ici j’y ai ajouté les TOOLS comme j’utilise VMWare pour ne pas avoir à le refaire à chaque fois, PUTTY et google chrome.

On effectue le sysprep en OOBE, en cochant la case « Généraliser » et en Arrêtant le système.

 

On redémarre ensuite la même machine sur le firmware.

 

Ici, on choisira de lancer la machine sur le EFI Network.

 

ATTENTION : votre serveur ici doit être allumé, et les deux machines doivent être sur le même réseau pour pouvoir communiquer.

 

Votre machine ira chercher l’image Capture.wim précédemment créée sur le serveur et lancer la création d’image (prenez garde à choisir le bon fichier BOOT pour ne pas relancer une installation. Si la capture et le boot ont le même nom)

*

Il faudra passer les premières étapes pour arriver sur cet écran :

Indiquer un emplacement local ou enregistrer la capture,

Cocher la case charger l’image sur un serveur de déploiement,

Bien indiquer le NOM DU SERVEUR ET NON LE NOM DU DOMAINE !

Puis sélectionner le groupe d’images dans lequel on veut ajouter notre capture.

 

Une fois fait il faudra entrer les informations du serveur sur lequel on veut charger l’image :

Nom d’utilisateur (admin du domaine)

Mot de passe (de l’admin du domaine)

 

L’image se copiera sur le serveur WDS. Une fois cette (longue) étape terminée, on peut éteindre notre machine virtuelle.
